# Feature: markdown heading and SEO escaping fix

## Goal

Fix Markdown note rendering where heading levels were rewritten incorrectly and body heading IDs did not match the index/TOC slugs. Also prevent SEO meta tags from leaking page excerpt text into the visible page when excerpts contain quoted HTML examples. Verify with the reported `项目代码解释/Fig4_B代码解释.md` page and focused renderer/web tests.

## Conclusion

Updated rendered heading anchor insertion to preserve the original `h1`-`h6` level and to reuse the `NoteRecord.headings` slug generated by the indexer. This keeps body heading IDs aligned with TOC and topbar links. Heading anchors are now added before embedded-note placeholders are restored, so embedded note headings are not processed a second time by the parent note.

Escaped SEO `<title>` text and all meta/link attribute values. The reported note excerpt includes `sprintf("<span ...>")`; before this fix, the double quote before `<span>` closed the meta `content` attribute and the remaining excerpt was rendered visibly above the app header.

Also corrected two source Markdown issues in the reported vault note: moved Chinese quotes outside bold markers in one sentence, and moved the final code-fence close to its own standalone line.
